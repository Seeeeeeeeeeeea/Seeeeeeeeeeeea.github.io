## 研究时间线

### 2025年5月：进入课题组，开始科研实习

2025年5月，我加入西安电子科技大学计算机科学与技术学院苗启广教授团队，开始进行科研实习。我的科研起点是参与“面部属性可编辑的说话头视频生成算法”相关研究。

在该项目初期，我主要负责论文展示图的制作以及项目主页展示视频的制作。同时，我在本地复现了相关方法，完成模型推理，并参与后续论文的比较分析工作。随着论文经历一系列转投和修改过程，我逐渐参与到更深入的论文修改与撰写工作中，并根据审稿人意见完成了多轮论文内容调整。

这一阶段对我而言是从“参与科研”到“理解科研流程”的起点。通过该项目，我学会了使用 Markdown 整理科研笔记，掌握了基于 LaTeX 的科技论文写作流程，并熟悉了使用 Zotero 进行文献管理和论文引用整理的方法。

### 2025年7月至8月：补充深度学习基础，深入理解扩散模型与说话头方法

2025年暑假期间，我集中学习了深度学习基础知识，观看了李沐和李宏毅的深度学习课程，系统补充了神经网络、深度学习训练机制以及生成模型相关基础。

在此期间，我也开始阅读领域内经典论文，例如 AniPortrait、Hallo 等开源说话头生成方法，并对前期参与的面部属性可编辑说话头视频生成算法进行了更深入的代码阅读。通过这一过程，我逐渐熟悉了 UNet 架构、扩散模型的基本原理，以及条件控制在生成任务中的作用方式。

这一阶段的核心意义在于，我不再只是执行具体实验或制作展示材料，而是开始理解模型结构、训练逻辑和方法设计背后的原因，为后续独立思考研究问题打下了基础。

### 2025年下半年：从纯关键点半身数字人生成出发，发现身份泄露问题

在进一步的科研实践中，我们最初希望研究基于纯关键点的跨身份半身数字人生成方法，对标 AnimateAnyone 和 MimicMotion 等方法，并以 AniPortrait 后半部分的 pose2video 模块作为基座方法进行实现。

然而，后续训练结果显示，当使用全量关键点进行跨身份驱动时，模型容易出现身份泄露和身份偏移问题。也就是说，驱动关键点不仅携带动作信息，还可能隐含原始驱动视频中的身份特征，从而干扰参考图所提供的身份信息。

基于这一现象，我们逐渐明确了一个核心问题：如果要实现具有泛化能力的半身数字人视频生成，必须解决“身份信息应该从哪里来、动作信息应该从哪里来”的责任划分问题。具体来说，可以有两条路线：

第一条路线是让生成视频与参考图保持身份对齐，即将驱动关键点中的身份信息主动对齐到参考图身份上；第二条路线是让驱动关键点本身尽可能不包含身份信息，从源头上减少关键点对身份的干扰。

### 2025年10月至12月：尝试通过 3D 表示对齐身份信息

第一条路线的核心思路是将驱动关键点的身份与参考图身份进行对齐。我们当时考虑的是“音频到 3D 模型，再到 2D 关键点，最后到视频生成”的技术路线。

受到 AniPortrait 的启发，我们进一步考虑通过 Audio2Mesh 的方式，将参考图身份融合到 3D 表示中，再转换为 2D 关键点。随后，我们找到了经典方法 EMAGE。该方法可以实现音频驱动关键点生成，并且关键点中的面部轮廓信息，也就是身份相关信息，可以由一组参数 betas 控制。原则上，如果能够从参考图中提取这组 betas，并将其替换到音频驱动生成的 pose 中，就有可能实现身份统一。

围绕这一思路，我开展了具体实现工作：在本地部署 OpenPose 以提取对应关键点，学习了经典 3D 人体建模方法 SMPL-X，并复现了人体体态估计相关方法，包括 SMPLify-X、SMPLer-X 和 BLADE 等。

但在实际操作过程中，我们发现该路线存在较高成本。我们的目标任务是半身场景下的数字人生成，而这些方法通常基于全身数据进行训练，生成的关键点也更适合全身人体场景。在半身数字人任务中直接使用时并不完全适配，可能需要修改网络架构，或者增加针对半身场景的标注数据重新训练，整体成本较大。因此，这一方向在实验验证后被暂时放弃。

### 2026年1月至5月：探索不携带身份信息的关键点，实现解耦式半身数字人生成

随后，我们转向第二条路线：使用尽可能不携带身份信息的关键点进行驱动。

根据前期实验，我们已经知道面部稠密关键点可能导致身份泄露。因此，最直接的思路就是逐步去掉面部稠密点，并观察不同关键点设置对生成结果的影响。我们依次进行了多组测试，包括去掉面部轮廓、去掉面部所有稠密点、去掉头部骨骼点等。

最终，受到 EchoMimicV2 和 V-Express 的启发，我们认为一种更合理的条件划分方式是：使用头部三个关键点控制头部朝向，使用手部关键点控制手部动作，使用音频控制面部动态，尤其是嘴部运动，而参考图则只负责提供身份信息。

这种设计的关键在于让不同条件各司其职：关键点不再包含明显身份信息，参考图可以自由切换，模型也不容易出现身份过度偏移，从而实现跨身份泛化生成。

与此同时，我们也观察到关键点作为强条件时，可能会压制音频对面部动态，尤其是嘴部区域的控制。针对这一问题，我们引入了条件丢弃操作：随机将带有头部三点的关键点替换为不带头部三点的关键点，使模型在训练过程中学习如何平衡头部结构控制与音频动态控制之间的关系。

为了支撑模型训练，我们进行了大量数据采集，覆盖头部视频和半身视频，并使用关键点检测方法以及 DOVER 美学检测方法对视频进行筛选。在此基础上，我们进一步开展了音频编码器相关实验，分别测试了 Whisper + linear projection、Wav2Vec2 + Q-Former、Whisper 裸接等多种方式，以探索最适合当前任务的音频注入方式。最终结果显示，Whisper 裸接方法对嘴部驱动效果更加有效。

综上，我们主要解决了两个问题。

第一个问题是，以 EchoMimicV2 为代表的方法在实现跨身份半身数字人泛化生成时，缺乏头部结构化控制条件，导致音频驱动的头部动作容易出现僵硬和抖动。我们通过引入头部三点关键点作为轻量结构控制信号，增强了头部姿态控制能力。

第二个问题是，半身数字人在多语言音频驱动场景下仍然存在不足。我们通过大量收集多语言数据，并结合不同音频编码方式实验，提升了模型在多语言语音驱动下的嘴部同步能力。

基于上述方法，我们可以实现跨身份数字人生成，也就是类似 AI 换脸的参考身份替换；也可以实现嘴部编辑，即在相同动作下更换说话语音；还可以实现动作编辑，即在同一参考身份下更换驱动关键点，从而生成不同动作表现的视频。

## 个人思考

在前期科研推进过程中，我遇到了不少问题。最明显的感受是，在思考创新点时，我常常会感到迷茫，不知道应该从哪里入手。后来我逐渐意识到，创新点并不是凭空想出来的，而是需要通过大量阅读论文、理解领域问题，并结合实验现象不断验证和沉淀出来的。

从论文中，可以获得其他方法对于问题的理解方式，也可以将相关领域的 insight 迁移到当前任务中；从实验中，则可以发现一些原本没有被清晰表达出来的现象。这些现象如果能够被解释、被验证，并进一步转化为方法设计，就可能形成真正有价值的创新点。

在这一过程中，我通过实践逐渐掌握了扩散模型的基本原理，也通过当前接触到的方法获得了很多启发。比如，相比于扩散模型中的 CFG 策略，它通常是在模型推理阶段放大某个条件的作用；而条件 Dropout 的思想更关注训练过程中的条件分配问题。它不是简单地在推理时加强某个条件，而是在训练时让模型学会不同条件之间应该如何分工。

结合我的实践来看，条件 Dropout 的作用就是权衡用于控制头部朝向的关键点和用于控制面部动态的音频之间的关系，从而实现条件解耦。这个方法虽然简单，但非常有效，也符合我对好方法“大道至简”的理解。它对后续多条件解耦控制任务也可能具有启发意义，尤其适用于多个条件之间存在冲突或控制权竞争的场景。

另外，我对扩散时间步这一方向的 insight 也非常感兴趣。在后续阅读相关文献时，我发现 T2I-Adapter 等文章提出，扩散模型在不同去噪时间步中具有不同的作用粒度：扩散早期更决定结构和运动，后期更偏向细节修复。类似观点在相关超分领域文章中也有所体现。此外，数字人相关方法 StableAvatar 在音频经过 Audio-Adapter 注入时，也引入了扩散时间步条件。

我认为这一发现非常有意思。扩散模型就像一个画家，在早期先勾勒出想要绘制图像的整体轮廓，虽然这个轮廓在人眼看来可能并不明显；而在后期，模型则不断补充纹理、细节和局部质量。

围绕这一现象，我也进行了实验验证。在正常训练得到的权重下进行推理时，我将 1000 个扩散时间步划分为早期、中期和后期三个阶段，并分别测试头部三点结构信息在不同阶段注入时的影响。实验结果显示，这一现象依旧成立：如果在扩散早期引入头部三点结构信息来引导头部朝向，生成结果几乎和全程引导没有明显差异；而如果只在中期或后期引入，则无法达到同样效果。

我认为这个发现可能会对后续基于扩散模型的条件可控生成带来较大启发。它说明条件控制不一定需要在整个扩散过程中持续作用，不同条件可能应该在不同时间步承担不同责任。结构类条件更适合在早期发挥作用，细节类条件可能更适合在后期发挥作用。如何根据条件类型设计时间步注入策略，可能是一个值得继续探索的问题。

此外，我也重新思考了“音频到动作、动作到视频”的技术路线。这个思路后来很快被同期工作如 CyberHost、VersaAnimator 等方法验证。我认为，相比于让音频直接作用在 pixel 或 latent 上，先由音频生成动作，再由动作驱动视频生成，可能更加有意思，也更贴近真实应用场景。

例如，在新闻电视台口播主持人这类场景中，用户不可能手动生成一长段关键点来驱动数字人，这样会非常麻烦。更合理的方式应该是由音频自动生成对应的关键点动作，再通过动作驱动视频生成。音频驱动动作这样的 pose generation 任务，也可能与具身智能存在交叉，例如机器人随着节拍跳舞、根据语音节奏做动作等场景。

另外，我也关注扩散时间步压缩问题。当前业界已经开始使用 DiT 架构进行扩散时间步压缩，并在一些场景下达到实时生成水平。但这类方法的缺点也很明显：算力需求较大，通常需要多卡协作才能实现实时推理。相比之下，我认为对更加轻量化的 UNet 架构进行扩散时间步压缩探索，同样具有应用价值。它可能不一定追求最高性能，但更适合在计算资源受限的场景中部署和使用。

## 兴趣方向

### 情感计算

我对情感计算比较感兴趣。当前主流数字人方法大多已经可以做到“有形”，也就是在外观、口型、动作等方面生成较为自然的结果。但如果对标具身智能中“情智兼备”的机器人，现有数字人在情感表达、情绪理解和人机交互方面仍然有较大提升空间。

因此，我认为情感数字人仍然是一个值得研究的方向。它不仅需要生成看起来像人的外观和动作，还需要理解语音、语义、情绪和场景，并在表情、语调、动作节奏等方面做出更加自然的反馈。

### 模型压缩

我也对模型压缩方向感兴趣。何凯明曾提到扩散模型从多步去噪压缩到一步生成的方向，我认为这个问题非常有意思。扩散模型的多步去噪机制虽然带来了强大的生成能力，但也导致推理成本较高。

当前这一方向其实还没有被广泛、成熟地实现，尤其是在高质量视频生成和数字人生成任务中，如何在保证生成质量的同时减少推理步数、降低显存占用和计算成本，仍然是一个很有价值的问题。

### 多模态控制

多模态控制是人工智能中的重要研究方向。真正智能的系统不应该只理解单一模态，而应该能够对语音、图像、视频、动作、文本等多模态信号进行联合理解，并做出合理反应。

在数字人生成任务中，多模态控制尤为重要。参考图、音频、关键点、文本、情绪标签等条件都可能参与生成过程。如何让这些条件各自承担清晰的责任，同时避免相互冲突，是我比较关注的问题。

### 3D重建

我也对 3D 重建方向感兴趣。3D 场景与具身智能、数字人、元宇宙等方向都存在交叉。相比二维图像或视频，3D 表示能够提供更强的空间结构信息，也更适合真实交互和物理环境理解。

未来如果数字人需要进入更复杂的三维空间，或者与真实世界、虚拟场景、机器人系统发生交互，那么 3D 重建和 3D 表征能力将会变得非常重要。
